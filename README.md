# 나의 독서 일기

읽은 책을 검색해서 기록하는 로컬 우선 웹앱입니다. 책 제목이나 ISBN으로 정보를 불러오고, 완독 날짜와 감상을 남겨 내 서재처럼 관리할 수 있습니다.

## 주요 기능

- 책 제목 또는 ISBN으로 도서 정보 검색
- 완독 날짜, 평점, 태그, 감상 기록
- 내 서재 목록, 상세 보기, 삭제, 수정
- 현재 베스트셀러 참고 보기
- JSON 내보내기와 가져오기
- 브라우저 `localStorage` 저장

## 프로젝트 구조

- `index.html`: 화면 구조와 PWA 메타 정보
- `styles.css`: 전체 UI 스타일
- `src/app.js`: 화면 상태, 이벤트, 저장 흐름
- `src/services/`: 도서 검색 API 연동
- `src/state/`: 기록 데이터 처리
- `src/ui/`: 카드와 목록 렌더링
- `server.js`: 정적 파일 서빙과 네이버 API 프록시
- `render.yaml`: Render 배포 설정

## 로컬 실행

1. `.env.example`을 참고해서 `.env`를 만듭니다.
2. 네이버 개발자 센터에서 발급받은 값을 넣습니다.
3. 서버를 실행합니다.

```bash
npm run dev
```

브라우저에서 `http://localhost:8000/`을 엽니다.

## Render 배포

1. 이 저장소를 GitHub에 푸시합니다.
2. Render에서 `New Web Service`를 선택합니다.
3. 저장소를 연결하고 `render.yaml` 설정을 사용합니다.
4. 환경 변수에 다음 값을 넣습니다.
   - `NAVER_CLIENT_ID`
   - `NAVER_CLIENT_SECRET`
5. 배포가 끝나면 Render가 준 공개 주소로 접속합니다.

## iPhone에서 확인

1. iPhone Safari에서 Render 공개 주소를 엽니다.
2. 공유 버튼을 누릅니다.
3. `홈 화면에 추가`를 선택합니다.
4. 아이콘처럼 추가된 앱을 실행하면 웹앱처럼 사용할 수 있습니다.

## 사용 흐름

1. 홈에서 `책 추가`를 누릅니다.
2. 책 제목이나 ISBN을 입력해 후보를 찾습니다.
3. 책을 선택하고 완독 날짜와 감상을 적습니다.
4. 저장한 뒤에는 `내 서재 보기`에서 목록과 상세 내용을 확인합니다.
5. 필요하면 JSON으로 백업하거나 복원합니다.

## 환경 변수

- `NAVER_CLIENT_ID`: 네이버 검색 API 클라이언트 ID
- `NAVER_CLIENT_SECRET`: 네이버 검색 API 클라이언트 시크릿
- `PORT`: 서버 포트, 로컬에서는 기본값 `8000`
